Identity Platform

Federated Authentication at Enterprise Scale

Quark · 2023–2024

🎯 The Challenge

  • Multiple external identity providers needed to act as a single federated source of truth.
  • High authentication latency (~2s) causing user experience degradation under load.
  • Third-party provider failures were causing cascading system-wide outages.

🛠️ The Solution

  • Designed a federated identity architecture using Keycloak SPI with external IdPs as the source of truth.
  • Introduced Redis-based caching to bypass redundant IdP calls, drastically reducing latency.
  • Built custom OIDC token mappers enabling dynamic authorization via JWT claims.
  • Implemented timeout, retry, and fallback strategies to maintain login availability under external failures.
